# Plan: Index Clojure Classpath Libraries (No JVM)

## Context

The LSP currently indexes only project source files (`src/`, `test/`). To provide completions, hover, and goto-definition for library symbols (e.g., `clojure.string`, `ring.core`, `reagent`), we need to index all library JARs on the classpath. The constraint is: **no JVM startup** — `clojure -Spath` is too slow for an LSP server to run on initialization.

---

## Part 1: Discovering the Classpath Without the JVM

### Best approach: Read `.cpcache/` (tools.deps cache)

When `clojure` CLI runs, it computes and caches the full classpath (including all transitive deps) in `.cpcache/<hash>.cp` as a plain colon-separated string. This file already exists if the user has ever run their project — **no JVM needed to read it**.

```
.cpcache/
  3847291abc.cp       ← plain text: "/home/user/.m2/repo/.../ring-core-1.9.6.jar:/home/user/.m2/..."
  3847291abc.libs     ← EDN: full dep graph with coordinates and paths
```

**Discovery strategy (in priority order):**
1. Find `.cpcache/*.cp` files in project root → read the most recently modified one
2. *(Future fallback)* Parse `deps.edn` and resolve POM files from `~/.m2/repository`

For **Leiningen** projects:
- `.lein-classpath` file (generated by `lein classpath`) or
- Read `project.clj` and resolve from `~/.m2` (harder — future work)

**Classpath parsing:** Use `std::env::split_paths()` — not manual colon-splitting — so it works correctly on both Unix (`:`) and Windows (`;`).

### Output
A `Vec<PathBuf>` of JAR paths and source directories from the classpath.

---

## Part 2: Efficiently Indexing Library JARs

JARs are ZIP files. Each JAR may contain `.clj/.cljs/.cljc` source files (in addition to `.class` bytecode). We can use the existing `extractor::extract()` on these source files.

### Key efficiency strategies

1. **Per-JAR disk cache** — The biggest win. Indexed symbols for an unchanged JAR should be stored to disk (`.clj-lsp/jar-cache/<jar-hash>.bin`) and loaded on subsequent starts without re-parsing.
   - Cache key: JAR path + file modification time (no need to hash contents)
   - Cache format: `bincode`-serialized structs (fast, compact)
   - Cache invalidation: if JAR mtime changes, re-index

2. **Background async indexing** — Start the LSP immediately using project-source index, then index JARs in a background task. Update the shared `Arc<Index>` when done. This avoids blocking `initialize()`.

3. **Parallel JAR processing** — Use `rayon::par_iter()` (already used for files) across the list of JARs.

4. **Fast early filtering** — Before parsing, scan JAR entries for `.clj`/`.cljs`/`.cljc` extensions. Skip pure-Java JARs immediately (most Java libs have no Clojure sources).

5. **Private symbol filtering** — For library JARs (not project source), skip `defn-` private functions and namespaces ending in `impl`/`internal`.

---

## Part 3: goto_definition for JAR Symbols

JAR entries are not real filesystem paths — `Url::from_file_path(sym.file)` breaks for them. The v1 approach:

- Add a `SymbolSource` enum to `Symbol`:
  ```rust
  pub enum SymbolSource {
      Project,
      Jar(PathBuf),  // path to the containing JAR
  }
  ```
- `goto_definition` returns `None` silently for `Jar` symbols. Completion and hover work correctly since they don't need a file path.
- v2 (future): implement a `jar://path/to.jar!/clojure/string.clj` URI scheme with a `textDocument/content` provider to serve decompressed source.

## Changes to Existing Files

### `src/index/mod.rs`
- Add `SymbolSource` enum (derives `serde::Serialize/Deserialize`)
- Add `source: SymbolSource` field to `Symbol`
- Add `serde` derives to `Symbol`, `NsMeta`, `DefKind` (needed for cache serialization)

### `src/index/scanner.rs`
- Add `index_classpath_jars(root, jars, index)` function:
  1. Filters to JAR files
  2. Checks cache for each JAR (load if fresh, skip re-parse)
  3. Re-indexes stale/missing JARs and saves cache
  4. Inserts all symbols into shared Index

### `src/handlers/definition.rs`
- Before building the `Location` response, check `sym.source`:
  - `SymbolSource::Project` → return `Location` as today
  - `SymbolSource::Jar(_)` → return `None` (no goto_definition in v1)

### `src/server.rs` — `initialize()` handler
```rust
// After spawning project source scan:
tokio::spawn(async move {
    let jars = classpath::discover(&root);
    scanner::index_classpath_jars(&root, jars, Arc::clone(&index)).await;
    client.log_message(..., "Library indexing complete").await;
});
```

### `Cargo.toml`
New dependencies:
- `zip = "2"` — for reading JAR files
- `bincode = "1"` — for fast cache serialization
- `serde = { version = "1", features = ["derive"] }` — for cache serialization

- Round-trip test: `save()` then `load()` and assert the entry is identical
- Cache-hit test: call `load()` with a matching mtime → returns `Some`
